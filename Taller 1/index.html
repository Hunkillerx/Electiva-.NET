ENSAYO .NET



.NET

Es importante entender como .Net a pesar de haber nacido como la respuesta de Microsoft al creciente mercado del desarrollo Web , como competencia a Java o a los numerosos frameworks basados en PhP, desarrollo una estrategia característica para el apadrinamiento de desarrolladores de diferentes lenguajes de programación con la implementación del CLR ( Common Language Runtime) que permite que código escrito en lenguajes adoptados por el framework como C#( lenguaje nativo para .NET ), VB, C++,Python, etc., puedan ser “traducidos” por el CLS ( Common Language Specification) para su correcto compilamiento dentro del CLR, por lo que rompió el paradigma de mono lenguaje para el uso de frameworks.

Además, en su plataforma de “Full Framework ” o .NET “Tradicional”, permite la creación de aplicaciones de todo tipo y ambiente, como: consola, escritorio web, móviles, tiende de Windows, etc. pero, con la característica principal de ejecutarse únicamente en ambientes Windows. Pero enfocado a cumplir objetivos claramente establecidos como:

  • Entorno coherente al desarrollo de programación orientada a objetos, donde el código se almacena y ejecuta de manera local, distribuida en internet o remota.

  • Proporcionar un entorno de ejecución de código que reduzca lo máximo posible la implementación de software y los conflictos de versiones.

  • Seguridad en la ejecución del código, aunque provenga de terceros o fuentes desconocidas.

  • Entorno de ejecución de código que elimine problemas de rendimiento al usar scripts o interprete de comandos.

  • Ofrecer al desarrollador una experiencia coherente entre soluciones diferentes, sea basada en Windows o Web.

  • Asegurarse que el código de .Net Framework se pueda integrar con otros tipos de código.



.Net Core

Pero en su constante búsqueda de asimilamiento dentro del mundo de los desarrolladores decidió la creación de .NET Core, un framework de código abierto y multiplataforma, nació como una nueva versión modular de .NET tradicional, aunque siendo un subconjunto de ella por lo que no tiene toda la funcionalidad del framework completo, aun así, se emplea para la creación de aplicaciones Web, de escritorio y móvil. Teniendo claros objetivos en mente desde su creación:

  • Ligera y componentizable, lo que permite que la aplicación se distribuya únicamente para su enfoque, no para la plataforma completa.

  • Multiplataforma, como característica principal permite que las aplicaciones creadas usando .Net Core funcionen tanto en Windows, como en Linux y Mac. Liberándolas de Microsoft.

  • Alto rendimiento, posee un desempeño más alto que la versión tradicional, vital para los entornos Cloud.



Xamarin

Dentro del aspecto de apoderamiento del Framework que ha estado implementando Microsoft para .Net se decidió adquirir e implementar Xamarin que nació como un proyecto Open Source que trataba de llevar .Net a todas las plataformas en la época en que Microsoft se negaba a sacar .Net de Windows. Fue ahí donde el mexicano Miguel de Icaza desarrollo Xamarin, construyendo sobre Mono toda la tecnología para la creación nativa de aplicaciones móviles en todas las plataformas (Windows Mobile, iOS y Android ). Cabe destacar que Xamarin utiliza C# como lenguaje nativo y las aplicaciones son migradas a la plataforma deseada.

Por todo esto y más, el mismísimo Bill Gates se jacta de que la plataforma .Net es el producto de software sobre el que mas se ha escrito y el mas premiado, por lo que no es difícil encontrar materia de sobra, así como una excelente documentación para iniciar y llevar a cabo el aprendizaje de la plataforma junto con todos sus componentes, lo que convierte a .Net como una prioridad de aprendizaje para cualquier desarrollador que desee hacer parte de una de las plataformas mas usadas en el entorno laboral así como aprender y usar el 7° lenguaje de programación mas usado en el mundo como lo es C#.
